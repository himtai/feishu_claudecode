## 1. 项目架构概览

### 1.1 核心组件

本项目采用模块化架构，将飞书集成与 Claude Agent 功能分离，主要包含以下核心组件：

1. **WebSocket 连接层** (`feishu_websocket.py`)
2. **消息处理层** (`async_message_handler.py`)
3. **Agent 执行层** (`execute_claude.py`, `persistent_agent.py`)
4. **会话管理层** (`session_manager.py`)
5. **文件传输层** (`file_uploader.py`, `file_downloader.py`)
6. **状态通知层** (`status_notifier.py`)
7. **工具集成层** (`feishu_tools.py`, `feishu_client.py`)

### 1.2 技术栈

- **飞书 SDK**: `lark_oapi` - 飞书开放平台 API
- **Claude SDK**: `claude_agent_sdk` - Anthropic Claude Agent SDK
- **异步框架**: `asyncio` - Python 异步 I/O
- **多线程**: `threading` - 并发处理
- **数据持久化**: JSON 文件存储

### 2.1 feishu_websocket.py (18KB)
**主入口文件 - WebSocket 长连接模式**

**核心功能：**
- 使用 WebSocket 长连接接收飞书消息（无需公网 IP 和 webhook）
- 处理多种消息类型：文本、图片、文件、富文本（post）
- 支持特殊命令：`/help`, `/status`, `/cancel`, `/new`
- 消息去重机制（避免重复处理）
- 自动下载图片和文件

**关键特性：**
- 解决嵌套 Claude Code 会话问题（删除 CLAUDECODE 环境变量）
- 支持交互式问题回答（用户选择选项）
- 后台线程处理异步任务
- 持久化 Agent 管理（保持上下文）

**消息处理流程：**
```
接收消息 → 去重检查 → 类型识别 → 下载资源 →
检查待回答问题 → 处理命令 → 调用 Agent → 后台执行
```

**代码质量：**

### 4.1 性能优势

   - 无需轮询，实时性高
   - 减少网络开销

   - 避免并发冲突

4. **上下文压缩**
   - 依赖 Claude SDK 自动压缩

### 4.2 性能瓶颈

1. **文件 I/O**
   - 会话数据频繁读写 JSON 文件
   - 建议使用数据库（SQLite）

2. **同步阻塞**
   - 部分文件操作是同步的
   - 建议使用 aiofiles

3. **内存占用**
   - 完整对话历史保存在内存
   - 建议定期持久化到磁盘
